UsageTracker 0.1.3 makes credential access and background-service restarts more reliable while giving each provider control over its setup experience.

## Highlights

- Avoid duplicate Keychain authorization prompts during overlapping account discovery and refreshes.
- Isolate and time-bound Keychain operations so a stalled credential request cannot wedge the daemon.
- Stop the old background service completely before app restarts, updates, or replacements continue.
- Show provider-defined setup controls consistently in onboarding and Settings.

## Installation

UsageTracker requires macOS 14 or newer.

```sh
curl --proto '=https' --tlsv1.2 -fsSL \
  https://github.com/joeymalvinni/usagetracker/releases/latest/download/install.sh | bash
```

The installer verifies published SHA-256 checksums, archive contents, and ad-hoc code-signature integrity. By default it installs:

- `UsageTracker.app` in `~/Applications`
- `usage` in `~/.local/bin`

> [!IMPORTANT]
> This release is ad-hoc signed and is **not notarized by Apple**. macOS may require you to approve UsageTracker manually the first time it opens. Follow the [Open Anyway instructions](https://github.com/joeymalvinni/usagetracker/blob/v0.1.3/docs/troubleshooting.md#opening-the-unnotarized-app) without disabling Gatekeeper globally.
