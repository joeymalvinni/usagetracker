UsageTracker 0.1.7 keeps usage visible while offline, adds a configurable activity grid, and improves app launch and provider sign-in behavior.

## Highlights

- Keep last-known usage visible during network outages with clear offline status in the app and CLI.
- Switch the activity view between the existing bar chart and a calendar-style activity grid.
- Open the menu bar popover reliably when launching or reopening UsageTracker.
- Copy provider sign-in links without the provider CLI opening a browser.

## Installation

The UsageTracker menu bar app requires macOS 14 or newer.

```sh
curl --proto '=https' --tlsv1.2 -fsSL \
  https://github.com/joeymalvinni/usagetracker/releases/latest/download/install.sh | bash
```

The installer verifies published SHA-256 checksums, archive contents, and ad-hoc code-signature integrity. By default it installs:

- `UsageTracker.app` in `~/Applications`
- `usage` in `~/.local/bin`

> [!IMPORTANT]
> This release is ad-hoc signed and is **not notarized by Apple**. macOS may require you to approve UsageTracker manually the first time it opens. Follow the [Open Anyway instructions](https://github.com/joeymalvinni/usagetracker/blob/v0.1.7/docs/troubleshooting.md#opening-the-unnotarized-app) without disabling Gatekeeper globally.
