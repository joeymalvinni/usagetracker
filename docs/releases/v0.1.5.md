UsageTracker 0.1.5 keeps collection running as a macOS background service, makes provider setup more private and guided, and improves Codex and Claude usage accuracy.

## Highlights

- Keep usage collection and the CLI available after closing the menu bar app through a per-user macOS LaunchAgent.
- Start with only Codex enabled, inspect other providers only after opting in, and copy provider sign-in links when needed.
- See Codex token activity and cost estimates from local session logs with cached input counted and priced correctly.
- See Claude scoped limits such as Fable while retaining five-hour and seven-day data when legacy fallbacks are needed.
- Avoid repeated Keychain authorization prompts while managed Claude sign-ins refresh stale cached credentials automatically.
- Build and run the daemon and CLI on macOS or Linux with the new Nix flake.

## Installation

The UsageTracker menu bar app requires macOS 14 or newer.

```sh
curl --proto '=https' --tlsv1.2 -fsSL \
  https://github.com/joeymalvinni/usagetracker/releases/latest/download/install.sh | bash
```

The installer verifies published SHA-256 checksums, archive contents, and ad-hoc code-signature integrity. By default it installs:

- `UsageTracker.app` in `~/Applications`
- `usage` in `~/.local/bin`

> [!IMPORTANT]
> This release is ad-hoc signed and is **not notarized by Apple**. macOS may require you to approve UsageTracker manually the first time it opens. Follow the [Open Anyway instructions](https://github.com/joeymalvinni/usagetracker/blob/v0.1.5/docs/troubleshooting.md#opening-the-unnotarized-app) without disabling Gatekeeper globally.
