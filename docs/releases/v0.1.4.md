UsageTracker 0.1.4 adds a provider-focused CLI and live local usage updates while improving Codex data, authentication recovery, alerts, installer feedback, and app presentation.

## Highlights

- Use the redesigned CLI with provider shortcuts like `usage codex`, plus new `summary`, `activity`, and scoped `status` views.
- See locally observed token and cost activity update promptly when supported provider data changes.
- View Codex rate-limit reset credits consistently across summary and detail views.
- Recover from Keychain authentication failures and locate Codex correctly when the app launches outside a shell.
- Avoid duplicate threshold notifications when provider reset times shift, with faster response processing throughout the app and CLI.
- Follow clearer installer progress and verification feedback, and enjoy a unified native glass popover.

## Installation

UsageTracker requires macOS 14 or newer.

```sh
curl --proto '=https' --tlsv1.2 -fsSL \
  https://github.com/joeymalvinni/usagetracker/releases/latest/download/install.sh | bash
```

The installer verifies published SHA-256 checksums, archive contents, and ad-hoc code-signature integrity. By default it installs:

- `UsageTracker.app` in `~/Applications`
- `usage` in `~/.local/bin`

> [!IMPORTANT]
> This release is ad-hoc signed and is **not notarized by Apple**. macOS may require you to approve UsageTracker manually the first time it opens. Follow the [Open Anyway instructions](https://github.com/joeymalvinni/usagetracker/blob/v0.1.4/docs/troubleshooting.md#opening-the-unnotarized-app) without disabling Gatekeeper globally.
