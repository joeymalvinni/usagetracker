The first public release of UsageTracker: a local macOS menu bar app and CLI for monitoring usage, limits, resets, and estimated costs across AI coding tools.

## Highlights

- Track Codex, Claude, OpenCode Go, and Grok from one menu bar app.
- Monitor provider limits, reset times, remaining capacity, and usage forecasts.
- Keep multiple Codex, Claude, and Grok accounts in isolated profiles.
- View account-wide provider data separately from local activity and cost estimates.
- Receive desktop notifications when authoritative quota is running low.
- Inspect the same data from the `usage` terminal command.

## Protocol and developer tooling

- Introduces socket API v3 with typed errors, combined state, provenance, and background refresh jobs.
- Includes generated request and response JSON Schemas.
- Adds complete CLI, configuration, security, privacy, troubleshooting, provider, and API documentation.

## Installation

UsageTracker requires macOS 14 or newer.

```sh
curl --proto '=https' --tlsv1.2 -fsSL \
  https://github.com/joeymalvinni/usagetracker/releases/latest/download/install.sh | bash
```

The installer verifies published SHA-256 checksums, archive contents, and ad-hoc code-signature integrity. By default it installs:

- `UsageTracker.app` in `~/Applications`
- `usage` in `~/.local/bin`

> [!IMPORTANT]
> This release is ad-hoc signed and is **not notarized by Apple**. macOS may require you to approve UsageTracker manually the first time it opens. Follow the [Open Anyway instructions](https://github.com/joeymalvinni/usagetracker/blob/v0.1.0/docs/troubleshooting.md#opening-the-unnotarized-app) without disabling Gatekeeper globally.

## Upgrade note

UsageTracker is still pre-1.0. Provider data is reproducible, so incompatible legacy UsageTracker database schemas may be reset instead of migrated. Configuration and credentials are preserved separately.
