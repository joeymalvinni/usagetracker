UsageTracker 0.1.2 adds secure in-app updates and shows you what changed after each update.

## Highlights

- Install newer stable releases directly from the dashboard.
- Verify the published installer checksum before updating the app in place.
- See a concise summary and release highlights after the updated app relaunches.
- Spot available updates with a clearer button and installation progress state.

## Installation

UsageTracker requires macOS 14 or newer.

```sh
curl --proto '=https' --tlsv1.2 -fsSL \
  https://github.com/joeymalvinni/usagetracker/releases/latest/download/install.sh | bash
```

The installer verifies published SHA-256 checksums, archive contents, and ad-hoc code-signature integrity. By default it installs:

- `UsageTracker.app` in `~/Applications`
- `usage` in `~/.local/bin`

> [!IMPORTANT]
> This release is ad-hoc signed and is **not notarized by Apple**. macOS may require you to approve UsageTracker manually the first time it opens. Follow the [Open Anyway instructions](https://github.com/joeymalvinni/usagetracker/blob/v0.1.2/docs/troubleshooting.md#opening-the-unnotarized-app) without disabling Gatekeeper globally.
