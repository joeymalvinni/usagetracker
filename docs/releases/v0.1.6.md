UsageTracker 0.1.6 adds comprehensive Cursor usage tracking and improves Codex resets, provider sign-in, and app updates.

## Highlights

- Track Cursor included plan limits, Auto/API lanes, Enterprise personal caps and team pools, legacy request quotas, and on-demand budgets.
- Review complete Cursor billing-cycle activity with individual events, per-day and per-model costs, and vendor-versus-metered comparisons.
- Keep cached Cursor sessions tied to validated accounts while refreshing authentication from Cursor.app during collection.
- See Codex rate-limit reset dates reliably again.
- Copy provider sign-in links when browser-based authentication needs manual follow-up.
- Avoid stale menu bar app processes when installing an update.

## Installation

The UsageTracker menu bar app requires macOS 14 or newer.

```sh
curl --proto '=https' --tlsv1.2 -fsSL \
  https://github.com/joeymalvinni/usagetracker/releases/latest/download/install.sh | bash
```

The installer verifies published SHA-256 checksums, archive contents, and ad-hoc code-signature integrity. By default it installs:

- `UsageTracker.app` in `~/Applications`
- `usage` in `~/.local/bin`

> [!IMPORTANT]
> This release is ad-hoc signed and is **not notarized by Apple**. macOS may require you to approve UsageTracker manually the first time it opens. Follow the [Open Anyway instructions](https://github.com/joeymalvinni/usagetracker/blob/v0.1.6/docs/troubleshooting.md#opening-the-unnotarized-app) without disabling Gatekeeper globally.
